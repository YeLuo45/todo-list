# TodoList V57 PRD: 任务依赖关系图 + 拓扑排序

## 1. 背景

V56 完成了 SmartPriorityEngine 智能优先级预测。V57 实现**任务依赖关系图**，支持依赖关系建模、DAG 可视化、拓扑排序，灵感来自 nanobot-design TaskRegistry。

## 2. 方向

**Direction G: 任务依赖关系图** — 任务间依赖关系建模 + 拓扑排序 + 循环检测 + 并行任务识别

## 3. 功能需求

### 3.1 TaskDependencyGraph 核心引擎
- `TaskDependencyGraph` 类：DAG（有向无环图）结构
- `addDependency(taskId, dependsOn)` — 添加依赖关系
- `removeDependency(taskId, dependsOn)` — 移除依赖关系
- `getDependencies(taskId)` — 获取任务的直接依赖
- `getDependents(taskId)` — 获取依赖该任务的其他任务

### 3.2 拓扑排序
- `topologicalSort(): Task[]` — Kahn 算法实现，返回任务执行顺序
- `getReadyTasks(): Task[]` — 当前无未完成依赖的任务（可立即执行）
- `getNextRunnable(pendingTasks): Task[]` — 给定待执行任务集，返回可执行的任务

### 3.3 循环检测
- `hasCycle(): boolean` — 检测是否存在循环依赖
- `getCycleTasks(): string[]` — 返回形成循环的任务 ID 列表

### 3.4 并行任务识别
- `getParallelBatches(): Task[][]` — 按层级分批，同一批内任务可并行执行
- `getMaxParallelism(): number` — 最大并行度（某层的任务数）

### 3.5 任务完成传播
- `markComplete(taskId)` — 标记任务完成，传播更新可执行状态
- `getCompletionImpact(taskId): string[]` — 任务完成会解除哪些任务的阻塞
