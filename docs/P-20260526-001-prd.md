# TodoList V54 PRD: MCP Client Retry + Transport Resilience + Tool Caching

## 1. 背景

V53 完成了离线优先 SQLite + MCP Client Bridge，V54 在此基础上增强连接可靠性和性能优化。

## 2. 方向

**Direction A: MCP Retry + Transport Resilience + Tool Caching**

## 3. 功能需求

### 3.1 MCP Client Retry Logic
- `MCPClient` 添加自动重试机制（指数退避）
- 连接失败时最多重试 3 次，间隔 1s → 2s → 4s
- `connect()` / `callTool()` 失败时触发重试

### 3.2 Transport Health Check
- 添加 `healthCheck()` 方法检测传输层是否可用
- HTTP transport 超时检测（5s）
- 定期心跳机制避免僵尸连接

### 3.3 Tool Result Caching
- `ToolCache` 类：基于 tool name + args hash 缓存结果
- TTL：5 分钟（可配置）
- 缓存大小限制：100 条（LRU 淘汰）
- 缓存失效：tool list 变化时清空

## 4. 技术方案

```
src/mcp/
  client.ts       — MCPClient 添加 retry + healthCheck
  tool-cache.ts   — NEW: ToolCache 类
  retry.ts        — NEW: retry utility (exponential backoff)
  types.ts        — 添加相关类型定义
```

- 覆盖率 ≥ 80%
